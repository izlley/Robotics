## 3. 결정 / 발견

### π0의 가장 중요한 새로움 (3가지)

1. **MoE single transformer**:
   - VLM weight set + Action expert weight set이 한 transformer 안에 공존
   - Token type별 deterministic routing (image/lang → VLM, state/action → action)
   - Self-attention layer에서만 상호작용
   - **이는 LLM의 Switch Transformer (learnable sparse routing)와 다른 형태** — dense + fixed routing

2. **Block-wise causal attention** + state KV cache:
   - 3 blocks로 attention 분리: img+lang / state / action
   - Block 사이 causal forward만 → VLM distribution shift 방지
   - State block 분리 덕분에 ODE 10 step에서 state KV reuse → inference ~5x 가속

3. **Pre/post training 명시적 분리**:
   - Pre: 다양한 robot, mixed quality, 10K hours
   - Post: 단일 task, high quality, 5-100 hours
   - 둘 다 필수 (Pre-only는 brittle, Post-only는 mistake recovery 못 함)
   - LLM의 base + SFT 패턴을 robotics에 최초 도입

### SmolVLA와의 핵심 차이 (paradigm은 같지만 구현은 다름)

| 차원 | π0 | SmolVLA |
|---|---|---|
| 총 params | 3.3B | 0.45B |
| 두 expert 상호작용 | MoE 안에 (single transformer) | Cross-attention (two transformers) |
| Action mask | bidirectional within | causal within |
| τ injection | MLP fusion | AdaLN |
| Pre-training data | 10K hours, 7 robots, 68 tasks | 23K episodes, SO-100 단일 |
| Open? | Closed (openpi 재구현) | Full open |

→ **π0가 paradigm 정립, SmolVLA가 open+lightweight으로 확장**.

### Cross-embodiment 18-dim padding trick

모든 robot의 action dim을 18로 통일 (가장 큰 robot 기준). 작은 robot은 zero-pad. 이미지 수도 max 3, 부족하면 mask out.

**LLM tokenizer padding의 robotics version**. 간단하지만 cross-embodiment 학습의 핵심 enabler.

## 4. 학습한 것

### 비자명한 발견

- **τ sampling이 image gen과 반대**: image gen은 "mean image" 학습이 hard → middle τ. Robot은 observation이 action을 강하게 제약 → "mean action" 학습이 hard → **low τ** (high noise) 강조. 이는 generative model의 task-specific tuning이 robotics에서 필요함을 보여주는 사례.

- **State KV cache 5x inference 절약**: 작은 architectural trick이지만 real-time control에 결정적. State는 ODE step 동안 안 변하므로 한 번 계산하고 reuse. 같은 발상이 multi-step inference 모델 전반에 적용 가능 (Track c new idea).

- **Pre/post training 분리의 robotics 의미**: LLM에서는 당연한 패턴이지만 OpenVLA 같은 모델은 단일 단계. π0가 이 LLM 패턴을 robotics에서 ablation으로 입증한 첫 사례.

- **Action expert dimension 차별화**: VLM expert (width 2048, mlp 16384) vs Action expert (width 1024, mlp 4096). 같은 transformer 안에서 서로 다른 width 사용 가능 (head_dim과 head 수 정렬만 필요). **이는 robotics에 특화된 inference-cost 최적화** — action expert는 10번 forward이므로 크기에 민감.

### LLM 엔지니어 관점 takeaway

π0의 모든 구성요소는 LLM 도구의 robotics 응용:
- MoE → Switch Transformer 변형
- Block mask → Prefix LM (PaLM, T5)
- KV cache → 표준 inference 최적화
- Flow matching → DiT의 robotics 응용
- Pre/post training → SFT pipeline
- High-level VLM planner → LLM agent

**robotics-specific으로 새로 배울 게 거의 없음** — RT-2/OpenVLA/SmolVLA에서 다룬 paradigm 그대로의 더 정교한 통합.

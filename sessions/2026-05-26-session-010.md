## 3. 결정 / 발견

### π0.5의 가장 중요한 발견 (4가지)

1. **Architecture는 π0와 거의 동일** — 결정적 변화는 **데이터·학습 recipe**.
   - PaliGemma 3B + Action expert 300M = 3.3B (같음)
   - τ injection만 다름 (MLP fusion → adaptive RMSNorm, DiT-style)
   - **→ Data and training is all you need (at this stage)**

2. **97.6% non-target-embodiment data**로 학습 가능:
   - MM (target mobile manipulator): 400h, **2.4%**
   - 다른 source: 97.6% (다른 robot, web data, subtask, verbal instruction)
   - 그럼에도 unseen real homes에서 10-15분 task 성공
   - → **Diversity beats quantity**. LLM의 Instruction Tuning 발견과 일치.

3. **Hybrid action representation 결정적 — 특히 language ability**:
   - π0 (flow matching only) = language following 약함
   - π0-FAST+Flow (hybrid but no co-training) = medium
   - π0.5 (FAST pre-train + Flow post-train + co-training) = highest
   - → **Discrete token pre-training (FAST)이 LLM-style language abilities 보존에 필수**

4. **Hierarchical inference는 same model에서 더 잘 작동**:
   - π0.5 own HL > GPT-4 HL > human HL oracle (놀라움!)
   - In-domain HL training이 외부 LLM zero-shot보다 우월
   - **Implicit HL** (학습엔 포함, 추론 안 함)도 explicit 거의 따라잡음 → multi-task spillover

### Real Home Evaluation의 중요성

본 프로젝트 정독 중 **처음으로 진짜 unseen homes에서 검증된 모델**:
- 3개 실제 가정집 (학습에 없음)
- Kitchen + bedroom multi-stage cleaning
- 10-15분 task duration
- → End-to-end VLA의 open-world generalization 첫 입증

### Verbal Instruction (VI)이라는 새로운 supervision modality

전문가가 robot에게 실시간으로 subtask를 말로 지시 → trained low-level policy로 실행 → 그 sequence가 HL training data.

- 11% data 비중인데 제거 시 크게 떨어짐
- LLM의 RLHF / Constitutional AI의 robotics 버전
- Human-in-the-loop subtask demonstrations

## 4. 학습한 것

### Architecture vs Data/Recipe innovation의 분리
- π0.5는 architecture 거의 안 바꾸고 data + recipe만 진화 → 큰 capability gain
- 이는 **데이터와 학습 방법론이 architecture만큼 (혹은 그 이상) 중요**한 단계임을 시사
- LLM 진화 (GPT-3 → GPT-4)에서도 같은 패턴

### Discrete vs Continuous representation의 보완성
- 둘이 trade-off가 아니라 **complement**:
  - Discrete (FAST): 학습 빠름, language semantic 학습 well-suited
  - Continuous (Flow matching): 추론 빠름, dexterous action에 강함
- Hybrid가 둘의 장점 결합

### Co-training Source의 분업
- ME + CE → manipulation generalization
- WD → unseen object 인식
- HL → high-level reasoning
- VI → subtask sequence learning
- 각 source가 다른 axis에 기여 → 모두 함께 있어야 완성

### Diagram-alignment 규칙 첫 적용 성공
- pi0.md에 비해 pi0_5.md는 markdown table 위주 + indent+arrow 사용
- ASCII art 사용 시 모두 ASCII single-width chars만 사용
- 사용자가 학습 시 alignment 문제 발견하면 즉시 피드백 가능

## 4. 학습한 것

### Curriculum 설계의 visibility
ROADMAP이 single source of truth라고 명시했지만, **세부 stack 선택은 paper summary 안에 묻혀 있었음**. 이건 다음을 시사:
- ROADMAP만 봐서는 "어떤 도구를 학습하면 모든 모델 cover 가능한지" 불명확
- Stack 선택 가이드를 ROADMAP 안에 명시적으로 두어야 함
- 이번 갱신으로 해결

### Open-source 생태계의 분파
같은 paradigm (flow matching VLA)이지만 LeRobot vs openpi 두 별도 fork:
- LeRobot: HuggingFace 주도, SmolVLA 친화적
- openpi: Physical Intelligence 자체, π0 시리즈 reference

향후 Track C (회사 RFM) 단계에서 둘 중 어느 stack을 base로 할지 결정 필요. **현 단계에서는 둘 다 평가**하는 게 합리적.

## 3. 결정한 것 / 발견한 것

### OpenVLA vs RT-2 핵심 차이
- Backbone: closed PaLI-X/PaLM-E → open Llama-2 7B
- Vision: 단일 ViT-22B → **DINOv2 + SigLIP fusion (600M)**
- Data: 130K single embodiment → **970K 22+ embodiment**
- Co-training: 50-66% web → **로봇만** (semantic gen에서 RT-2-X에 일부 뒤짐, but manipulation 자체는 우위)
- Inference: cloud TPU 1-3 Hz → **RTX 4090 6 Hz, int4 7GB 가능**
- Fine-tuning: 미연구 → **LoRA r=32 + Full FT 동등**

### 비표준 학습 결정 3가지 (저자가 ablation으로 입증)
1. **Vision encoder unfreeze** (VLM 표준은 freeze) — robot domain의 fine-grained spatial 학습 필요
2. **27 epoch** (LLM 표준 1-2 epoch) — robot data가 narrow하고 reuse 가능
3. **224×224 충분** (384×384 동등 성능, 3x 시간) — VLA에서 high-res는 도움 안 됨

### 사소하지만 큰 영향의 디테일
- **Quantile-based discretization**: outlier 1% 제거로 effective granularity ~100x
- **BridgeData zero-action filter**: 매 demo 첫 step의 all-zero action을 제거. RT-2-X가 freezing 보이는 한 원인 → +20pp 차이 일부 설명

## 4. 학습한 것

### LLM 도구의 robot 응용 — 확장된 매핑
RT-2가 보여준 "action을 token으로" 외에 OpenVLA가 추가한 LLM 도구들:
- **LoRA r=32 / target=all-linear** → VLA fine-tuning 표준 recipe
- **int4 quantization** → 성능 손실 없이 메모리 ½
- **FSDP + HuggingFace AutoModel** → 인프라 그대로 재사용

LLM 엔지니어가 robotics에 진입할 때 사실상 새로운 학습 거의 없음.

### 후속 모델 분기점
OpenVLA의 한계가 후속 모델의 motivation:
- Single image → multi-view (π0+)
- Discrete bin ceiling → continuous (π0, Diffusion Policy)
- 6 Hz limit → action chunking (ACT, π0)
- 7B 무거움 → 경량화 (SmolVLA)
- 22 embodiment 한정 → cross-embodiment (X-VLA, Octo)
- Static policy → RL self-improvement (π★0.6)

→ 다음 정독 어느 방향이든 OpenVLA를 baseline reference로 가능.
